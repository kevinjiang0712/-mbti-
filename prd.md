1. 背景与目标
1.1 背景
面向“家里养猫、把猫当孩子”的年轻用户，通过可观察行为题库，为猫咪生成 16 型人格（CBTI），输出标签化结果与简短描述，满足“给猫贴标签”的娱乐与情绪价值需求。
1.2 MVP目标（上线可用）
- 3 个页面跑通闭环：首页 → 测试页 → 结果页
- 全部本地数据（题库、结果库、计分逻辑），不接后端、不接分享、不做埋点。
1.3 非目标（MVP不做）
- 分享海报、转发、好友互动
- AIGC 图片生成/上传猫照片
- 账号体系、云开发存储、历史记录
- 数据埋点、A/B、商业化、订阅、广告

---
2. 用户与使用场景
2.1 目标用户
- 18–35 养猫人群，偏“情感投射/拟人化”用户
- 单身/不婚倾向用户（非必须，但契合）
2.2 核心场景
用户打开小程序，用 1–2 分钟完成测试，获得“我家猫是什么人格类型”的结论与简短解读。

---
3. 核心体验与信息架构
3.1 页面结构
- 首页（Home）
- 测试页（Quiz）
- 结果页（Result）
3.2 关键路径
进入小程序 → 点击开始测试 → 逐题选择 → 提交 → 输出结果

---
4. 功能需求（按页面）
4.1 首页（Home）
4.1.1 功能点
- 展示产品标题与一句话价值主张
  - 标题：猫咪 CBTI 人格测试
  - 副标题：用 12 个问题测出你家猫的“人格类型”
- 主 CTA 按钮：开始测猫
- 可选：简短提示（不影响流程）
  - “请按猫在多数情况下的表现选择”
4.1.2 交互
- 点击“开始测猫”进入测试页，默认从第 1 题开始，清空上次测试状态。
4.1.3 异常
- 无。

---
4.2 测试页（Quiz）
4.2.1 功能点
- 显示当前题目（单选）
- 显示进度：当前题号 / 总题数
- 选项选择后自动进入下一题（减少操作）
- 最后一题选择后，显示“查看结果”按钮（或自动跳转结果页）
4.2.2 数据与规则
- 题目数量：12 题（MVP固定）
- 每题两个选项，每个选项对应一个维度得分（E/I、S/N、T/F、J/P）
- 维度题目分配：每个维度 3 题
- 出结果规则：某维度某字母得分 ≥ 2 即取该字母，拼成 4 字母类型（例：INTJ）
4.2.3 状态管理
- currentIndex：当前题索引
- scores：四维累计分数结构，例如
  - E, I, S, N, T, F, J, P（初始化为 0）
- answers（可选）：记录每题选择，用于返回上一题（MVP可不做返回）
4.2.4 交互细节
- 单选点击即选中（高亮）并：
  - 若非最后一题：currentIndex + 1
  - 若为最后一题：展示“查看结果”按钮
- “查看结果”跳转到结果页，并传递计算出的 type
4.2.5 异常处理
- 用户未选择就试图下一步：MVP无“下一步”按钮则天然避免；若保留按钮则需 toast 提示“请选择一个选项”。

---
4.3 结果页（Result）
4.3.1 功能点
- 展示结果类型与人格称号
  - 例：INTJ 冷面霸总猫
- 展示标签（3–5 个）
- 展示简短描述（100–140 字）
- 提供 2 个按钮：
  - “再测一次”（回到测试页，清空状态）
  - “返回首页”（回到首页）
4.3.2 数据
- resultsMap：16 型结果文案映射（本地静态 JSON/TS 文件）
4.3.3 异常
- 未拿到 type 或映射缺失：
  - 默认 fallback：显示“未识别类型”，提供“再测一次”。

---
5. 计分与类型计算（功能规格）
5.1 计分键值（建议）
- 使用统一 scoreKey：E/I/S/N/T/F/J/P
- 每选择一次对相应键 +1
5.2 类型计算算法（确定性）
- EI：E >= 2 ? 'E' : 'I'
- SN：S >= 2 ? 'S' : 'N'
- TF：T >= 2 ? 'T' : 'F'
- JP：J >= 2 ? 'J' : 'P'
- type = EI + SN + TF + JP

---
6. 内容资产（MVP范围）
- 题库：12 题（固定题库）
- 结果库：16 型（固定文案）

7. 视觉与交互设计规范（核心配置：神秘塔罗牌 Design System）

7.1 颜色规范 (Color Palette)
- 主背景色 (Background)：暗夜星空渐变。
  - 起点色 (Top)：`#1A1A2E` (深邃午夜蓝)
  - 终点色 (Bottom)：`#4A306D` (柔和暗紫)
- 主题点缀/发光色 (Primary/Accent)：用于强调、按钮发光。
  - 魔法金：`#F9DC5C`
  - 荧光紫：`#D580FF`
- 文本色 (Typography Colors)：
  - 大标题/高亮：`#FFFFFF` (纯白，可附加 `0 2rpx 4rpx rgba(0,0,0,0.5)` 阴影增强可读性)
  - 正文/副标题：`#DED0E6` (浅灰紫，保证暗背景下的阅读体验)
  - 提示词/未选中态文字：`#8C7A9E` (暗紫灰)
- 卡片/透明底板色 (Surface)：
  - 基础面板：`rgba(255, 255, 255, 0.1)` (配合毛玻璃模糊度 `blur(20px)`)
  - 高亮面板（选中/按下）：`rgba(255, 255, 255, 0.15)`

7.2 字体规范 (Typography)
- 字体系列 (Font Family)：优先系统无衬线黑体（`PingFang SC`, `Helvetica Neue`），数字和特殊英文字符优先尝试圆体（如 `Arial Rounded MT Bold`）增加可爱感。
- 字号层级 (Font Sizes)：
  - 特大标题 (Display)：`40rpx`，字重 `800` (大标题、结果页名称)
  - 一级标题 (H1)：`36rpx`，字重 `600` (页面标题或卡片标题)
  - 正文 (Body)：`28rpx`，字重 `400`
  - 辅助说明 (Caption)：`24rpx`，字重 `400`
- 行间距 (Line Height)：大段文本统一使用 `1.6` 增加呼吸感。

7.3 圆角、阴影与边框规范 (Shape & Elevation)
- 圆角 (Border Radius)：全面屏除尖锐棱角。
  - 小组件 (标签、输入框)：`16rpx`
  - 中组件 (选项卡片、小弹窗)：`24rpx`
  - 大组件 (塔罗主牌、大容器)：`40rpx`
- 阴影规范 (Box Shadow)：
  - 基础柔和投影（悬浮感）：`0 8rpx 24rpx rgba(0, 0, 0, 0.2)`
  - 魔法发光阴影（强调）：`0 0 32rpx rgba(213, 128, 255, 0.4)` (荧光紫) 或金光 `rgba(249, 220, 92, 0.4)`
- 边框规范 (Border)：
  - 常规卡片包边（高光边缘）：`1px solid rgba(255, 255, 255, 0.2)`
  - 选中状态/高光状态：`1px solid #F9DC5C` (魔法金描边)

7.4 页面布局规范 (Layout)
- 屏幕结构 (Screen Structure)：采用“沉浸式全屏布局”，取消微信原生顶部灰色导航栏（使用自定义导航栏），让星空渐变延伸至系统状态栏底部。
- 安全距离 (Safe Area)：左右内容边距统一保持 `40rpx`，避免贴边；底部主按钮避开安全区（适配 `env(safe-area-inset-bottom)`），距底部至少 `32rpx`。
- 纵向留白 (Spacing)：采用大比例呼吸留白，标题与内容间最小间距 `32rpx`。

7.5 组件设计规范 (Components)
- 按钮规范 (Buttons)：
  - 形状：全圆角胶囊形（高度一半为 `border-radius`）。
  - 主按钮（Primary）：宽厚饱满（高度 `96rpx` 或 `100rpx`），背景使用发光色渐变，内嵌微弱内阴影制造立体果冻质感，无边框。
  - 次按钮（Secondary）：背景为纯透明，保留 `1px` 半透明白色边框和白色文字。
- 卡片规范 (Cards)：
  - 问卷选项卡片：高度 `120rpx`，居中对齐文字。默认透明度 `0.08` 白色底加高光边框；选中时卡片附带点击缩放动效（Scale: `0.95`），边框变魔法金色，触发设备轻机震动。
  - 塔罗主卡片（结果页的核心资产）：比例如 3:4，占据屏幕中心黄金比例。必须搭载 CSS “流光闪过”或“悬浮呼吸”特效。
- 输入区规范 (Input Areas - 扩展预留)：
  - 背景：`rgba(0, 0, 0, 0.2)` (加深底色以凸显其中的发光文字)。
  - 交互：输入框获取焦点时（Focus），边框高亮为荧光紫色。

7.6 视觉资产规范 (Assets)
- 图标规范 (Icons)：
  - 风格特征：抛弃单调线框，采用 2.5D 微拟物风格 或 带有粗外发光的双色线性图标（底色+高光点缀色）。
  - 图形要求：所有转角均为圆润处理（Cap & Join 均为 round）。
- 插画规范 (Illustrations)：
  - 画风定位：不要死板的纯平面矢量，需呈现微 3D 和果冻感局部高光（参考上文发出的 Catzy 图）。
  - 构图元素：重点表现魔法阵、星辰轨迹、塔罗牌阵，与猫咪融合。
  - 人格神态：猫咪神态需根据 MBTI 做出极致的差异化表达（如星星眼、黑眼圈、炸毛等），插画需导出为透明底色 WebP 格式以便融入星空背景。
